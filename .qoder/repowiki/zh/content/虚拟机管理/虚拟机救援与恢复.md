## 项目结构
救援与恢复功能由“HTTP 层 → 任务队列 → 服务层 → Libvirt/工具链”的分层架构组成。关键入口与职责如下：
- 路由层：注册 /api/vm/:name/rescue 与 /api/vm/:name/password/reset 等 API
- 处理器层：校验参数、提交异步任务、返回任务 ID
- 任务队列：统一调度与进度上报
- 服务层：实现具体逻辑（救援、密码重置、快照恢复）
- Libvirt/工具链：与虚拟化后端交互（virsh、virt-customize、virt-win-reg 等）

## 详细组件分析

### 救援模式启动与关闭流程
- 启动流程要点
  - 强制关机并等待至关机状态
  - 读取并保存原始配置（磁盘总线、网卡类型、引导顺序）
  - 将磁盘总线切换为兼容模式、网卡切换为通用型号
  - 挂载救援 ISO 并设置引导顺序为光驱优先
  - 开机进入救援环境
- 关闭流程要点
  - 强制关机并等待
  - 弹出救援 ISO
  - 加载原始配置并恢复磁盘总线/网卡/引导顺序
  - 开机回到正常模式
  - 清理临时配置文件

```mermaid
flowchart TD
Start(["开始"]) --> CheckState["检查虚拟机状态"]
CheckState --> |运行中| ForceOff["强制关机并等待关机"]
CheckState --> |已关机| SaveCfg["保存原始配置"]
ForceOff --> SaveCfg
SaveCfg --> SwitchBus["切换磁盘总线为兼容模式"]
SwitchBus --> SwitchNic["切换网卡为通用型号"]
SwitchNic --> MountISO["挂载救援 ISO"]
MountISO --> SetBoot["设置引导顺序为 cdrom 优先"]
SetBoot --> StartVM["开机进入救援模式"]
StartVM --> End(["结束"])
subgraph "关闭流程"
StopStart["开始"] --> StopForceOff["强制关机并等待"]
StopForceOff --> EjectISO["弹出救援 ISO"]
EjectISO --> LoadCfg["读取原始配置"]
LoadCfg --> RestoreBus["恢复磁盘总线"]
RestoreBus --> RestoreNic["恢复网卡类型"]
RestoreNic --> RestoreBoot["恢复引导顺序"]
RestoreBoot --> StopStartVM["开机回到正常模式"]
StopStartVM --> Cleanup["清理临时配置文件"]
Cleanup --> StopEnd(["结束"])
end
```

### 快照恢复策略
- 内部快照：直接使用 virsh snapshot-revert 恢复，必要时自动 resume
- 外部快照：关闭虚拟机、回切到快照创建时的磁盘状态、修复权限、设置当前快照标记并重启

```mermaid
flowchart TD
S1["选择快照"] --> CheckType{"内部快照？"}
CheckType --> |是| RevertInternal["virsh snapshot-revert 恢复"]
RevertInternal --> ResumeCheck{"恢复后是否暂停？"}
ResumeCheck --> |是| AutoResume["virsh resume"]
ResumeCheck --> |否| Done["完成"]
CheckType --> |否| CloseVM["关闭虚拟机"]
CloseVM --> BuildOverlay["构建恢复 overlay"]
BuildOverlay --> FixPerm["修复磁盘访问权限"]
FixPerm --> SetCurrent["设置当前快照标记"]
SetCurrent --> StartVM["启动虚拟机"]
StartVM --> Done
```

## 性能考虑
- 救援与密码重置均为异步任务，避免阻塞 API 响应
- 救援流程中对虚拟机状态轮询等待关机，建议在大规模并发时配合限速与重试策略
- 快照恢复涉及磁盘重建与权限修复，建议在低峰期执行并预留足够磁盘空间
- 使用进度回调提升可观测性，便于前端展示与用户感知

## 结论
该救援与恢复体系通过“异步任务 + 分层服务 + 钩子解耦”的设计，在保障安全性的同时提供了高可靠性的系统故障恢复能力。结合快照与离线修复手段，能够覆盖密码重置、系统修复、紧急排障等典型场景。建议在生产环境中配合严格的权限控制、高风险验证与完善的日志审计，持续优化任务调度与资源预留策略。
